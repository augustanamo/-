    主要的几个命令

    下载所需项目
    git clone https://github.com/CKTim/BlueTooth.git（https://github.com/CKTim/BlueTooth.git替换成你之前复制的地址）
    git # 拉取远程分支到本地
    新建文件夹
    git init# 初始化

    git clone git@github.com:XXXX.git # 下载项目到本地

    git fetch origin develop # （develop为远程仓库的分支名）

    git checkout -b dev(本地分支名称) origin/develop(远程分支名称)
    在本地创建分支dev并切换到该分支

    git checkout dev #切换分支

    git pull origin develop(远程分支名称)

    更改代码前
    git pull # 从远程更新代码
    git status # 查看当前仓库的状态

    本地更改后更新到git
    git add . 将工作区的修改提交到暂存区
    git commit -m ‘’ 将暂存区的修改提交到当前分支
    git push 将本地代码更新到远程分支上

    其他
    git reset # 回退到某一个版本
    git stash # 保存某次修改
    git reflog # 查看历史命令
    git diff # 查看修改
    git log # 查看提交历史
    git revert # 回退某个修改

    created 和 mounted 的区别

    created:在模板渲染成html前调用，即通常初始化某些属性值，然后再渲染成视图。

    mounted:在模板渲染成html后调用，通常是初始化页面完成后，再对html的dom节点进行一些需要的操作。

    $refs用于操作dom写在mounted里

    从输入URL到页面展示，中间发生了什么？

    1.在浏览器输入URL

    2.浏览器进行DNS解析URL为IP地址

    3.浏览器与ISP通信，通过DNS（可以理解为数据库）查找到域名对应的IP，发给浏览器

    4.TCP连接在发送数据之前建立客户端与服务端的通道进行发送数据

    5.TCP进行三次握手，客户端发送请求，服务器响应表示同意，客户端再次发送表示成功连接

    6.我们和服务器进行连接以后，浏览器发送HTTP请求报文{请求行（请求方法、请求地址、HTTP协议版本）、请求头部（浏览器信息，键值对组成）、空行、请求数据}给服务器

    7.服务器收到HTTP请求报文后进行处理并响应，响应报文{状态行（200请求成功，3XX重定向，404资源未找到，500服务器错误）、请求头部（浏览器信息，键值对组成）、空行、请求数据}

    8.浏览器接收到的HTML页面转换为DOM，如遇到CSS外链发出请求资源然后构建CSSDOM，两者结合成为渲染树

    9 .一旦浏览器加载完HTML涉及到的所有资源，页面最终会渲染在浏览器窗体里，并关闭与服务器的连接。
